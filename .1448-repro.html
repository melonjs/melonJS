#1448 — WebGLRenderer.createPattern repeat-mode parity

Source image is a 64×64 tile: orange top stripe (8 px), blue left stripe (8 px), dark grey background.

Two createPattern calls over that ONE source: top band uses "repeat-x", left band uses "repeat-y".

WebGL renderer

What to look for

PASS (post-fix): top band tiles horizontally — orange stripe runs across the full width, blue stripe appears only at the start of each tile column. Left band tiles vertically — blue stripe runs down the full height, orange stripe appears only at the top of each tile row.

FAIL (pre-fix): both bands show the SAME wrap mode — whichever createPattern call ran last wins. Typically you’d see the top band rendering with repeat-y (vertical stripes only) or the left band showing repeat-x (horizontal tiling). Either way one of the two looks wrong.

How this is rendered

const horiz = app.renderer.createPattern(source, "repeat-x");
const vert  = app.renderer.createPattern(source, "repeat-y");

// inside a custom Renderable's draw():
renderer.drawPattern(horiz, 0,   0,   640, 64);   // top band
renderer.drawPattern(vert,  0,   80,  64,  240);  // left band
